Ricos is a stylable component, which means you can apply colors scheme & font properties to your Ricos UI.

The theming API provides the ability to simply & consistently customize the style of your plugins, toolbars, and content.

```tsx
theme?: {
  palette?: PaletteColors | PalettePreset;
  customStyles?: RicosCustomStyles;
  parentClass?: string;
}
```

## Coloring

Ricos' coloring mechanism is manifested in almost every element that is displayed.

The structure is based on 3 color properties (aka `Palette`):

- Text Color
- Action Color
- Background Color

Check this [Theme Playground](http://rich-content-storybook-master.surge.sh/?path=/story/ricos--ricos-theme) to understand the influence of each color on the plugins, toolbars and modals.

You can also modify specific elements' colors (like text headers / links). See [Custom Styles](./Theming#custom-styles) API below.

### Palette

```jsx
const palette = {
  bgColor: '#0E092B',
  textColor: '#FFFFFF',
  actionColor: '#D6FF00',
};
// ...
<RicosEditor theme={{ palette }} plugins={[pluginHashtag()]} />;
```

<div className="card" style={{ backgroundColor: '#0E092B' }}>
  <RicosEditorWithMobile
    content={createContent('This is a #hashtag, see it has a unique color.')}
    placeholder={'Type here!'}
    theme={{
      palette: { bgColor: '#0E092B', textColor: '#FFFFFF', actionColor: '#D6FF00' },
      parentClass: 'card',
    }}
    plugins={[pluginHashtag()]}
  />
</div>

#### Palette Presets

Pick a palette preset by its name. Currently supports 'darkTheme' only.

```jsx
<RicosEditor theme={{ palette: 'darkTheme' }} />
```

## Custom Styles

Ricos allows customizing several font and color properties.
The `font-family` of textual content within Ricos is inherited from its wrapping component, while other typography properties like `font-size` are defined within Ricos.
Overriding these properties is done by setting the `customStyles`.

```tsx
const customStyles: RicosCustomStyles = {
  h2: {
    fontFamily: 'Times',
    fontSize: '26px',
    color: 'orange',
    fontStyle: 'italic',
    textDecoration: 'underline',
  },
  h3: {
    fontFamily: 'Tahoma',
    fontSize: '20px',
    color: 'purple',
    textDecoration: 'none',
  },
  quote: {
    lineHeight: '24px',
    fontWeight: 'bold',
    color: 'brown',
  },
};

// Example 1
<RicosEditor theme={{ customStyles }} />;

// Example 2
<RicosEditor />;
```

<div className="card card3" style={{ backgroundColor: '#0E092B' }}>
  <RicosEditorWithMobile
    content={customTypography}
    placeholder={'Type here!'}
    theme={{
      parentClass: 'card3', 
      customStyles: {
      h2: {
        fontFamily: 'Times',
        fontSize: '26px',
        color: 'orange',
        fontStyle: 'italic',
        textDecoration: 'underline',
      },
      h3: {
        fontFamily: 'Tahoma',
        fontSize: '20px',
        color: 'purple',
        textDecoration: 'none',
      },
      quote: {
        lineHeight: '24px',
        fontWeight: 'bold',
        color: 'brown',
      },
    }}}
  />
</div>
<div className="card card4" style={{ backgroundColor: '#0E092B' }}>
  <RicosEditorWithMobile
    content={customTypography}
    placeholder={'Type here!'}
    theme={{ parentClass: 'card4' }}
  />
</div>

Current customizable properties are:
`h2`, `h3`, `h4`, `h5`, `h6`, `p`, `quote`, `link`, `hashtag`, `button`.

and the accepted CSS properties are:
```jsx
{
  fontSize,
  fontFamily,
  fontWeight,
  fontStyle,
  textDecoration,
  lineHeight,
  minHeight,
  color,
}
```

:::note
`button` supports only the `color` field atm
:::

:::note
`quote` also supports `borderColor` for its indented quote border effect
:::

## parentClass

If you plan to apply different palettes for multiple Ricos instances living next to each other (like in this page, for example), you'll have to specify `parentClass`.
Otherwise, you can ignore this field.

`parentClass` should refer to a parent's css className that contains your ricos instance.
With this defined, dynamic styles will be wrapped under **".parentClass"**, which will increase CSS specifity.

```jsx
const darkOne = { bgColor: '#224455', textColor: '#FFFFFF', actionColor: '#FF66F5' };
const darkTwo = { bgColor: '#1C191C', textColor: '#D2CDD2', actionColor: '#FE6148' };
//...
<div className='card1'>
  <RicosEditor theme={{ parentClass: 'card1', palette: darkOne }} />
</div>
<div className='card2'>
  <RicosEditor theme={{ parentClass: 'card2', palette: darkTwo }} />
</div>
```
