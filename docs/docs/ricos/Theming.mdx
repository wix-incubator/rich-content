Ricos is a stylable component, which means you can apply colors scheme & font properties to your Ricos UI.

The theming API provides the ability to customize the style of your plugins, toolbars, and the content itself - with simplicity & consistency.

## Coloring

Ricos' coloring mechanism is manifested in almost every element that is displayed.

The structure is based on 3 color properties (aka `Palette`):

- Text Color
- Action Color
- Background Color

You are able to define these colors in the `palette` property. Make sure to check out our [Theme Playground](http://rich-content-storybook-master.surge.sh/?path=/story/ricos--ricos-theme)!

You can also modify specific elements' colors (like text headers / links). See [Custom Styles](./Theming#custom-styles) API below.

### Palette

```jsx
const palette = {
  bgColor: '#0E092B',
  textColor: '#FFFFFF',
  actionColor: '#D6FF00',
};
// ...
<RicosEditor theme={{ palette }} plugins={[pluginHashtag()]} />;
```

<div className="card" style={{ backgroundColor: '#0E092B' }}>
  <RicosEditorWithMobile
    content={createContent('This is a #hashtag, see it has a unique color.')}
    placeholder={'Type here!'}
    theme={{
      palette: { bgColor: '#0E092B', textColor: '#FFFFFF', actionColor: '#D6FF00' },
      parentClass: 'card',
    }}
    plugins={[pluginHashtag()]}
  />
</div>

## Font

By default, the `font-family` of the textual content within Ricos is inherited from outside (i.e container / page styling), while other typography properties like `font-size` were set by us.
which you can now customize via the Custom Styles API below:

### Custom Styles

As of consumers requests, you're able to customize several textual elements of Ricos in terms of font and color.

```tsx
const customStyles: RicosCustomTheme = {
  h2: {
    fontFamily: 'Times',
    fontSize: '26px',
    color: 'orange',
    fontStyle: 'italic',
    textDecoration: 'underline',
  },
  h3: {
    fontFamily: 'Tahoma',
    fontSize: '20px',
    color: 'purple',
    textDecoration: 'none',
  },
  quote: {
    lineHeight: '24px',
    fontWeight: 'bold',
    color: 'brown',
  },
};

// Example 1
<RicosEditor theme={{ customStyles }} />;

// Example 2
<RicosEditor />;
```

| h3?      | CustomTextualStyle | Textual Style of element |
| h4?      | CustomTextualStyle | Textual Style of element |
| h5?      | CustomTextualStyle | Textual Style of element |
| h6?      | CustomTextualStyle | Textual Style of element |
| p?       | CustomTextualStyle | Textual Style of element |

| Name            | Type                            | Short Description  |
| --------------- | ------------------------------- | ------------------ |

| lineHeight?     | CSSProperties['lineHeight']     | CSS Property Value |
| minHeight?      | CSSProperties['minHeight']      | CSS Property Value |

## parentClass (optional)

If you plan to apply different palettes for multiple Ricos instances living next to each other (like in this page, for example), you'll have to specify `parentClass`.
Otherwise, you can ignore this field.

`parentClass` should refer to a parent's css className that contains your ricos instance.
With this defined, dynamic styles will be wrapped under **".parentClass"**, which will increase CSS specifity.

```jsx
const darkOne = { bgColor: '#224455', textColor: '#FFFFFF', actionColor: '#FF66F5' };
const darkTwo = { bgColor: '#1C191C', textColor: '#D2CDD2', actionColor: '#FE6148' };
//...
<div className='card1'>
  <RicosEditor theme={{ parentClass: 'card1', palette: darkOne }} />
</div>
<div className='card2'>
  <RicosEditor theme={{ parentClass: 'card2', palette: darkTwo }} />
</div>
```
